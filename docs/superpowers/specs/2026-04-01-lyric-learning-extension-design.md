## 一、项目概述

### 1.1 项目名称
`lyric-learning-extension`

### 1.2 项目描述
Chrome 浏览器扩展，帮助用户学习英文歌曲。通过控制歌词逐句重复播放，让用户能够充分理解每句歌词的发音和含义。

### 1.3 目标平台
- Chrome 浏览器
- QQ 音乐 Web 版（https://y.qq.com）

---

## 二、核心功能

| 功能 | 描述 |
|------|------|
| 重复次数设置 | 用户在 Popup 设置每句歌词重复播放次数（1-10） |
| 自动计数跳转 | 扩展自动检测歌词切换，计数未达标时自动回跳 |
| 手动跳转 | 支持快捷键手动跳转上一句/下一句 |
| 状态显示 | Popup 显示当前歌词、已播放次数等信息 |

---

## 三、技术方案

### 3.1 实现方式
纯 DOM 监听：注入 content script 监听 QQ 音乐歌词区域的 DOM 变化，通过模拟点击控制播放。

### 3.2 技术栈
- Manifest V3
- TypeScript
- 原生 CSS（无框架）

### 3.3 核心交互流程

```
用户播放歌曲
    ↓
content script 检测歌词 DOM 变化
    ↓
歌词切换 → 读取当前歌词文本 → 检查该句重复计数
    ↓
计数 < 设定值 → 自动点击"上一句"（倒回去重播）
计数 >= 设定值 → 计数归零 → 等待下一句
```

## 五、模块设计

### 5.1 Popup 模块
- 用户设置重复次数（1-10）
- 显示当前歌词状态
- 无需保存按钮，实时生效

### 5.2 Content Script 模块

| 文件 | 职责 |
|------|------|
| `lyric-detector.ts` | 监听 QQ 音乐歌词 DOM 变化，检测当前播放到哪一句 |
| `counter.ts` | 管理每句歌词的重复计数 |
| `play-controller.ts` | 控制播放/暂停、跳转上一句/下一句 |

### 5.3 Background 模块
- 作为 popup 和 content script 之间的消息中转

### 5.4 Shared 模块
- 共享类型定义（消息类型、状态类型）
